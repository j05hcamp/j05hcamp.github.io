Mobile RFID Scanner — Portable Fixed-Reader System

Role: Concept dev, industrial design, embedded integration Scope: Mobile fixed-reader, offline ops, FOD prevention Outcome: $2M+ savings via loss prevention & efficiency

Originally conceived in 2019 to extend enterprise asset tracking capabilities beyond networked zones, the Mobile RFID Scanner redefined what portable inventory control could be. Built from the ground up—starting with an 80/20 frame and laser-formed acrylic housing—the design has evolved through seven major iterations, each improving form factor, power management, and system integration. The latest version features a custom 3D-printed chassis, modular electronics bay, fanless micro-PC, and 7" touchscreen interface, supporting variable voltage (12–24V) and six-hour battery operation. The device enables rapid, offline audits, live asset tracking, and real-time tool recovery—bridging the gap between fixed infrastructure and field mobility.

Affinity Flight is an interactive web-based animation exploring narrative storytelling through digital media. The project tells the story of a stranded space traveler finding their way home by following the sound of their family's song—demonstrating that emotional connection transcends physical distance.

Developed the complete project from initial concept through final deployment: created detailed storyboards, hand-drew all character and environment assets, and built the interactive experience using Adobe Animate. Technical execution included frame-by-frame animation, scene composition, timing and pacing control, and cross-platform web deployment.

Enterprise Account Management System — AI-Augmented Development

Role: Product strategy, full-stack development, system architecture Scope: Multi-tier account management with role-based access control Outcome: Functional enterprise application, pilot-ready in 11 working days

Built a functional enterprise account management system in 11 working days through strategic AI orchestration. Originally commissioned as an internal tool for managing customer accounts and action items, the project evolved into a comprehensive demonstration of AI-augmented rapid development capabilities.

The application manages hierarchical customer relationships across five organizational levels (Customer → Site → Program → Project → Task) with sophisticated role-based permissions supporting five distinct user types: Admin, Sales Agent, Coordinator (Supervisor), Coordinator, and Project Manager. Each role has carefully designed access controls and workflow permissions tailored to real-world team coordination scenarios.

Technical Architecture: Modern full-stack implementation using React + Vite frontend, Express.js backend, and PostgreSQL database. Features include recursive health scoring algorithms that aggregate status metrics from tasks up through the entire organizational hierarchy, multi-stage approval workflows for quality control, comprehensive audit logging, real-time notifications, and custom authentication without third-party dependencies.

The system implements complex business logic including automatic permission inheritance, cross-hierarchy task assignment, review-before-completion workflows, and customer satisfaction tracking—all documented in detailed technical specifications created alongside development.

  • 5-level hierarchical organization structure (Customer/Site/Program/Project/Task)
  • 5-tier role-based permission system with granular access controls
  • Recursive health scoring with parent-child status aggregation
  • Multi-stage task approval workflows
  • Comprehensive audit trail and activity logging
  • Real-time dashboard KPIs and health monitoring
  • Custom authentication and user management

Takeaway: This project validates that AI-augmented development, when combined with systematic problem-solving and clear requirements definition, can compress traditional development timelines while maintaining enterprise-grade architecture and functionality. The experience established foundational knowledge in modern web development practices and software engineering standards.

Process

01
Discover & Define

Walk the floor, map the workflow, define constraints and success criteria (what "done" means in the real world).

02
Model & Prototype

Prove it small and sturdy. Validate reads, ergonomics, and serviceability. Convert to manufacturable CAD and materials.

03
Integrate & Align

Define API requirements, map UI touchpoints, and specify operator feedback signals. Coordinate with software engineers to close the loop with audits and accountability.

04
Deploy & Improve

Ship what's durable, document what matters, train teams, capture feedback, and iterate for measurable gains.

Principles

Ownership over hand-offs.
Carry projects across the line. Reduce friction between siloed teams.
Operators are the users.
Every system is built to serve the people who use it daily. Designs should remove friction and provide immediate, intuitive feedback—whether through physical layout, interface cues, or equipment function.
Make it maintainable.
Fast to service, easy to train, and built from parts you can actually get.
